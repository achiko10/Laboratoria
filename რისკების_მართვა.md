# რისკების მართვის გეგმა

საგრანტო აპლიკაციაში მნიშვნელოვანია დაანახოთ, რომ გააზრებული გაქვთ პროექტის თანმდევი რისკები და ფლობთ მათი პრევენციის მექანიზმებს.

| რისკის დასახელება | რისკის ალბათობა | გავლენა პროექტზე | რისკის მართვის ან პრევენციის სტრატეგია (Mitigation) |
|---|---|---|---|
| **ტექნიკური რისკი:** მასწავლებლებს/სკოლებს არ აქვთ ძლიერი ინტერნეტი პლატფორმის გამოსაყენებლად. | მაღალი | საშუალო | პლატფორმის სიმულაციები დაწერილია **pure HTML/JS/CSS** კოდზე (გარე ბიბლიოთეკების გარეშე), რაც უზრუნველყოფს მათ წარმოუდგენელ სიმსუბუქეს. ფრაგმენტული ჩატვირთვა ნიშნავს, რომ ცალკეული გვერდები სწრაფად მუშაობს 3G ინტერნეტის ან Edge-ს პირობებშიც. შესაძლებელია Offline (ჩამოტვირთვადი) ვერსიის შემუშავება სკოლების კომპიუტერებისთვის. |
| **კურიკულუმის რისკი:** სიმულაცია არ ემთხვევა სკოლის პროგრამის ეტაპებს. | საშუალო | მაღალი | პროექტის საწყის ეტაპზე ინტენსიური კონსულტაციები გაივლება საგნის მეთოდოლოგებთან (ფაზა 1), რათა მასალა მკაცრად დაექვემდებაროს ეროვნულ სასწავლო გეგმას (ESG). |
| **ჩართულობის (ადაპტაციის) რისკი:** მასწავლებლებს არ სურთ ახალი ტექნოლოგიების გამოყენება გაკვეთილზე. | მაღალი | მაღალი | ტრენინგ-მოდულები იქნება მაქსიმალურად გამარტივებული. პლატფორმას აქვს ინტუიტიური UI (User Interface) — ქართულენოვანი ინსტრუქციებითა და მინიმალური ტექნიკური ბარიერით. |
| **ბიუჯეტის გადახარჯვის რისკი:** დეველოპმენტის ხანგრძლივობის გაზრდა ან ჰოსტინგის ხარჯების მოულოდნელი მატება მოთხოვნის გამო. | დაბალი | საშუალო | Cloud ინფრასტრუქტურა ოპტიმიზირებული იქნება დინამიურ ზრდაზე (Scalable). ბიუჯეტში ჩადებულია 5% გაუთვალისწინებელი ხარჯებისთვის, ხოლო ტექნიკური ძრავა (96 მოდული) უკვე დასრულებულია. კოდი ძირითადად Client-Side ხასიათისაა, რაც ამცირებს Cloud სერვერის დატვირთვას. |
| **ტექნიკური შეფერხება სასწავლო პროცესში (Server Downtime)** | დაბალი | მაღალი | სერვერი დაჰოსტილი იქნება უსაფრთხო და სტაბილურ პლატფორმებზე (მაგ. Amazon Web Services). შექმნილი იქნება სარეზერვო (Backup) ასლები და Load Balancing სისტემა ტრაფიკის სამართავად. |
